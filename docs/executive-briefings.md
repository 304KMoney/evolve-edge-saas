# Executive Briefings

Executive briefings are structured customer deliverables derived from completed audit reports.

## Source Of Truth

- The briefing is generated by the Next.js backend from validated report data.
- The briefing does not call OpenAI, LangGraph, n8n, Stripe, or HubSpot.
- The `Briefing` table stores the durable briefing summary and structured sections.

## Routes

- Report page CTA: `/dashboard/reports/[reportId]`
- Briefing view: `/briefings/[id]`
- Markdown export: `/api/briefings/[id]/export`

## Database

Migration:

- `packages/db/prisma/migrations/20260429163000_briefings/migration.sql`

Model:

- `Briefing.reportId` is unique so each report has one canonical briefing.
- `Briefing.organizationId` is used for tenant-scoped access checks.
- `Briefing.structuredSections` contains the presentation-ready sections.

## Manual QA

1. Complete an audit report.
2. Open the report detail page.
3. Confirm `View Executive Briefing` appears only when the report is downloadable/finalized.
4. Open the briefing route.
5. Confirm sections render: context, posture, findings, impact, actions, plan, recommendations, closing note.
6. Confirm markdown export downloads.
7. Confirm a user from another organization cannot access the briefing.
8. Confirm the briefing content is derived from report fields and does not expose raw AI payloads.
